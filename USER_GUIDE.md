# دليل استخدام SIM-NET

## 🚀 البداية السريعة

### 1. تشغيل النظام

**لـ Windows:**
```bash
# الطريقة 1: استخدام ملف التشغيل
python launcher.py

# أو
Start.bat
```

**لـ Linux/Mac:**
```bash
# الطريقة 1: استخدام ملف التشغيل
python3 launcher.py

# أو
./Start.sh
```

### 2. فتح المتصفح
بعد التشغيل، افتح المتصفح على:
```
http://localhost:8080
```

### 3. تسجيل الدخول

استخدم أحد الحسابات التالية:

| اسم المستخدم | كلمة المرور | الدور |
|-------------|------------|-------|
| admin | admin123 | مدير |
| engineer | engineer123 | مهندس |
| viewer | viewer123 | مشاهد |

## 📱 شرح الواجهة

### لوحة التحكم (Dashboard)

تعرض لوحة التحكم:
- **إحصائيات سريعة**: إجمالي الأجهزة، المتصلة، غير المتصلة
- **خريطة الأجهزة**: مواقع الأجهزة على الخريطة
- **آخر التنبيهات**: التنبيهات الحديثة
- **آخر الأنشطة**: السجلات الأخيرة
- **رسوم بيانية**: حالة الأجهزة

### إدارة الأجهزة (Devices)

#### إضافة جهاز جديد:
1. اضغط على زر "إضافة جهاز"
2. أدخل المعلومات المطلوبة:
   - IP Address
   - Hostname
   - نوع الجهاز (Router/Switch/Server/Firewall)
   - الشركة المصنعة
   - بيانات الاتصال (اختياري)
3. اضغط "حفظ"

#### Ping جهاز:
1. اضغط على زر الـ Ping بجانب الجهاز
2. انتظر النتيجة

#### حذف جهاز:
1. اضغط على زر الحذف (متاح للـ Admin فقط)
2. أكد الحذف

### المراقبة (Monitoring)

#### بدء المراقبة:
1. اضغط على "بدء المراقبة"
2. النظام سيراقب الأجهزة تلقائياً

#### Ping للكل:
1. اضغط على "Ping للكل"
2. انتظر النتائج

### الأمان (Security)

تعرض صفحة الأمان:
- **مشاكل الأمان المفتوحة**
- **الهجمات المكتشفة**
- **مستوى المخاطر**

### الأداء (Performance)

#### عرض أداء جهاز:
1. اختر الجهاز من القائمة
2. ستظهر رسوم بيانية للـ CPU والذاكرة
3. جدول بيانات الأدى يظهر في الأسفل

### تحليل AI (AI Analysis)

#### استخدام Chatbot:
1. اكتب مشكلتك في حقل النص
2. اضغط إرسال أو Enter
3. سيرد عليك الـ AI باقتراحات

**أمثلة على الأسئلة:**
- "interface is down"
- "high cpu usage"
- "slow network"
- "security alert"
- "vlan problem"

#### عرض تحليل Packets:
- يعرض التحليلات السابقة
- مستوى الخطورة
- نوع التهديد

### التقارير (Reports)

#### توليد تقرير:
1. اضغط على "توليد تقرير"
2. انتظر التوليد
3. سيتم تحميل التقرير

#### تصدير JSON:
1. اضغط على "تصدير JSON"
2. سيتم تحميل ملف JSON

### السجلات (Logs)

#### عرض السجلات:
- جميع السجلات تظهر في الجدول
- يمكن التصفية حسب الخطورة

#### تصفية السجلات:
1. اختر نوع السجل من القائمة
2. اضغط "تحديث"

### المستخدمين (Users) - للـ Admin فقط

#### إضافة مستخدم:
1. اضغط على "إضافة مستخدم"
2. أدخل:
   - اسم المستخدم
   - كلمة المرور
   - الدور (Admin/Engineer/Viewer)
   - البريد الإلكتروني
3. اضغط "حفظ"

## 🎨 الأدوار والصلاحيات

### Admin (مدير)
- ✅ جميع الصلاحيات
- ✅ إدارة المستخدمين
- ✅ حذف الأجهزة
- ✅ تعديل الإعدادات

### Engineer (مهندس)
- ✅ إضافة/تعديل الأجهزة
- ✅ مراقبة الشبكة
- ✅ تشغيل الأتمتة

### Viewer (مشاهد)
- ✅ مشاهدة Dashboard
- ✅ عرض الأجهزة
- ✅ عرض التقارير
- ❌ إجراء تغييرات

## 🔧 المهام الشائعة

### إضافة Router جديد
1. اذهب إلى "الأجهزة"
2. اضغط "إضافة جهاز"
3. أدخل:
   - IP: 192.168.1.1
   - Hostname: Router-Main
   - Type: Router
   - Vendor: Cisco
4. اضغط "حفظ"

### مراقبة جهاز
1. اذهب إلى "المراقبة"
2. اضغط "بدء المراقبة"
3. ستظهر حالة الجهاز

### تشغيل Ping
1. اذهب إلى "الأجهزة"
2. اضغط زر الـ Ping بجانب الجهاز
3. انتظر النتيجة

### الحصول على مساعدة من AI
1. اذهب إلى "تحليل AI"
2. اكتب مشكلتك في Chatbot
3. اضغط Enter

### توليد تقرير
1. اذهب إلى "التقارير"
2. اضغط "توليد تقرير"
3. سيتم تحميل التقرير

## ⚠️ استكشاف الأخطاء

### مشكلة: لا يمكن تسجيل الدخول
**الحل:**
- تأكد من صحة اسم المستخدم وكلمة المرور
- جرب الحسابات الافتراضية
- تحقق من تشغيل Backend

### مشكلة: لا تظهر الأجهزة
**الحل:**
- تأكد من تشغيل Backend
- اضغط على "تحديث"
- تحقق من قاعدة البيانات

### مشكلة: لا يعمل الـ Ping
**الحل:**
- تأكد من تشغيل Backend
- تحقق من اتصال الشبكة
- جرب Ping يدوياً من CMD

### مشكلة: لا تظهر الخريطة
**الحل:**
- تأكد من اتصال الإنترنت
- حاول تحديث الصفحة
- تحقق من Console Browser

### مشكلة: Backend لا يعمل
**الحل:**
```bash
# تأكد من تثبيت المتطلبات
pip install -r requirements.txt

# تأكد من عدم وجود عملية أخرى تستخدم المنفذ 5000
# Windows:
netstat -ano | findstr :5000

# Linux/Mac:
lsof -i :5000
```
